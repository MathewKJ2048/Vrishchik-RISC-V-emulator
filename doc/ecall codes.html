Ecall codes

The environment may be invoked by the use of the ecall command. This is useful for tasks like writing output to and reading input from the console. The ecall codes used are unique to the Vrishchik environment. The ecall code is stored in $a2

Code Purpose Arguments Return
0 terminate none none
1 read word none $a0
2 read byte (character) none $a0 (low)
3 read string (ASCII) $a0 stores start memory address, $a1 stores buffer size (in bytes) memory[$a0]
4 print register $a0 stores value to print,
$a1 stores base to print the value in,
$a3 is 0 if the value is to be treated as signed and not 0 otherwise		 none
5 print character $a0 (low) stores the character to be printed in ASCII none
6 print byte $a0 stores start memory address,
$a1 atores the base to print the value in,
$a3 is 0 if the value is to be treated as signed and not 0 otherwise		 none
7 print half $a0 stores start memory address,
$a1 atores the base to print the value in,
$a3 is 0 if the value is to be treated as signed and not 0 otherwise		 none
8 print word $a0 stores start memory address,
$a1 atores the base to print the value in,
$a3 is 0 if the value is to be treated as signed and not 0 otherwise		 none
9 print string $a0 stores the start memory address,
$a1 stores the length of the string to be printed		 none

If the argument for base is 0, base ten is assumed by default. If the length of the string to be printed is -1, the characters are printed till the null character is reached.
